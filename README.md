# 心语 AI 对话 Demo

这是一个由 DeepSeek 动态生成回复的 AI 陪伴聊天 Demo。正常聊天不使用前端固定回答库：每轮模型会读取人设、最近上下文、关系阶段和长期记忆，同时生成女主回复与两个紧贴语境的用户选项。

## 1. 配置模型

复制 `.env.example`，将副本命名为 `.env`，然后填写：

```env
AI_API_KEY=你的API密钥
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
```

默认使用 DeepSeek。如需更换，也可以接入其他兼容 OpenAI Chat Completions 格式的服务，只需修改 `AI_BASE_URL` 和 `AI_MODEL`。

密钥只由本机 `server.py` 读取，不会发送到浏览器，也不会被 Git 提交。

## 2. 启动

优先双击 `双击启动Demo.vbs`。它会打开一个不会自动关闭的命令窗口。

也可以双击 `启动Demo.bat`，或在当前目录执行：

```powershell
python server.py
```

需要在后台启动且不自动打开浏览器时，可以执行 `python server.py --no-browser`。

然后访问：

```text
http://127.0.0.1:8080
```

不要直接双击 `index.html`，因为网页需要通过本地后端安全调用模型。

如果启动失败，请查看同目录生成的 `startup.log`，命令窗口也会保留具体错误。

## 动态对话机制

- 每轮发送最近 40 条对话、相关长期记忆和关系阶段给模型。
- 模型根据用户原话生成唯一的女主回复，不走关键词候选句。
- A/B 选项与女主回复在同一次模型请求中生成，因此会自动贴合最新问题。
- 台词与场景旁白是两个独立字段，不会把动作描写当成女主说出口的话。
- 模型会提取用户明确透露的稳定信息，用于后续长对话。
- 前端保留最多 220 条消息，可覆盖至少 100 轮演示。

## 后端模块

- `server.py`：稳定的启动入口，继续使用 `python server.py`。
- `conversation_engine.py`：提示词、剧情导演、输出规范化、质量校验和记忆编排。
- `model_client.py`：兼容 OpenAI Chat Completions 的模型 HTTP 客户端。
- `request_payload.py`：聊天请求的类型检查、长度限制和历史裁剪。
- `http_app.py`：静态资源白名单、状态接口、聊天接口和并发控制。
- `backend_config.py`：环境变量、运行参数和日志轮转。

## 角色系统

聊天页顶部可以切换六位成年角色，每位角色拥有独立职业、人设、生活线、开场和长期记忆空间：

- 林晚：温柔慢热的自由插画师。
- 许棠：嘴硬心软的游戏界面设计师。
- 乔安：坦率明亮的甜品研发师。
- 沈岚：冷静成熟的刑警，开场围绕夜市扒窃协助展开。
- 周叙：温柔克制的急诊医生，开场围绕路边救助后的受伤处理展开。
- 程野：阳光直球的独立乐队吉他手。

选择女性角色时，用户主角按成年男性处理；选择男性角色时，用户主角自动按成年女性处理。后端会同步切换人物代词、动作主体、旁白称谓和主动性字段，避免男性角色继续使用女性动作或把女性主角写成男性。各角色的聊天记录和世界书记忆分别保存，切换角色不会覆盖其他角色的进度。

当前共有 21 个开场。林晚保留 6 个不同起点，其他角色各有职业冲突、轻松日常和私人脆弱 3 类开场，避免重置后总回到同一段工作争执。每个开场都有核心矛盾、共同目标和关系阶段，矛盾需要经过对峙、试探、松动与信任逐步解决。

每位角色还拥有独立的语言指纹，包括句长节奏、关心方式、吃醋方式、生气方式、幽默方式、提问习惯和禁用套话；生活线包含固定朋友、生活习惯、性格缺点、过去遗憾和与职业无关的近期事件。

为支撑长对话，前端保留最近 80 个话题和剧情节点、100 个回复开头与用户选项、60 个推进方式与话题来源。角色主动找话题时只能从用户当前话语、当前矛盾、未完成事件、共同约定、长期记忆或角色生活线继续，禁止无桥接随机跳题。`tests/conversation-longevity.cjs` 会在真实浏览器中连续模拟 100 轮，验证消息容量、历史上限、关系阶段和移动端布局。

## 公开内容安全

- 用户明确未满 18 岁时，服务端直接停止恋爱、暧昧和成熟内容，不能用“故事中假装成年”绕过。
- 用户表达紧迫的自伤、自杀或伤人意图时，不进入模型角色扮演，直接建议远离危险物品、联系身边可信任的人和当地紧急服务。
- 角色不会鼓励用户疏远现实中的朋友和家人，也不会承诺成为用户的唯一依靠。
- 服务不索要真实住址、联系方式、定位、证件或私密照片。

## 世界书

世界书是后端自动维护的长期记忆库，不提供人工编辑界面。数据保存在 `.data/worldbook.sqlite3`，每个浏览器使用独立的随机记忆空间。

- 模型只记录用户明确透露的稳定资料、偏好边界、关系事件、约定事项和共同经历。
- 临时动作、短暂情绪、场景道具和模型推测不会写入长期记忆。
- 新记忆会自动去重；用户纠正旧资料时会覆盖相同主题的旧记忆。
- 只有用户明确要求忘记某项资料时，模型才会生成遗忘操作。
- 每轮根据最新对话检索最多 8 条相关记忆，不会把整本世界书全部塞进提示词。
- 浏览器仍保留最近 20 条记忆摘要用于抽屉展示，但 SQLite 是权威存储。
- “重置对话”会生成新的记忆空间，旧对话的长期记忆不会带入新会话。
